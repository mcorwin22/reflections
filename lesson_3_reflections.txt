When would you want to use a remote repository rather than keeping all your work local?

To keep a backup of your work, should anything happen to the local files.

Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with your remote repository?

Some changes may have come from other users and could impacct your work locally in an unexpected way.

Describe the differences between forks, clones, and branches. When would you use one instead of another?

Forks create a new repository on your account, copying from the original.
Clones copy a repository from GitHub into your local repository.
Branches diverge from the master path, allowing you to create new changes without affecting the original.

What is the benefit of having a copy of the last known state of the remote stored locally?

You may not want to merge the current version of the file on GitHub.

How would you collaborate without using Git or GitHub? What would be easier, and what would be harder?

The group would probably work on separate tasks. Or at different times to not interfere.
It would be easier to not have to do commands, etc. It would be harder to work efficiently.

When would you want to make changes in a separate branch rather than directly in master? What benefits does each approach have?

Making a separate branch is useful when you want to make changes without affecting the master branch.
The benefits of having a separate branch are conserving the master branch.
The benefits of leaving pushes in the master branch is convenience.
